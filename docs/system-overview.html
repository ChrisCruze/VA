System at a Glance

The system is orchestrated as a sequence of interoperable services. Meetings and source material are captured, structured, and converted into publishable artifacts with audit trails.

Each block represents a managed integration. Airtable acts as the system of record for inputs, checklists, outputs, and distribution.

End-to-End Workflow

  1. Capture: Otter ingests meeting recordings and shares transcripts/highlights.
  2. Normalize: Zapier stores the conversation into Airtable Inputs, tagging owners, projects, and timestamps.
  3. Orchestrate: Google Opal (or custom orchestrator) kicks off the appropriate checklist run, invoking specialized agents.
  4. Produce: Agents create structured artifacts (briefs, drafts, assets) that are stored in Airtable and Google Docs.
  5. Distribute: Approved content moves through distribution automations into CMS and social channels.
  6. Measure: Run metrics, time spent, and publishing data flow back into dashboards for visibility.

Resilience & Monitoring

  • Error handling: Failed steps set the Run status to Blocked and alert the assignee with context and logs.
  • Version control: Prompts, artifacts, and runs carry version numbers to track evolution of the playbook.
  • Security: API keys remain in secrets managers; transcripts are sanitized prior to external API calls.
  • Governance: Human approval steps gate distribution to ensure compliance and final QA.
