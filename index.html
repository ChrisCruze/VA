Tools & agents

The recommended stack for capture, orchestration, and publishing.

Modular layers keep the signal high. Every tool has a defined job: capture truth, automate processing, produce artifacts, or publish outcomes.

Capture & transcription

  • Otter for meeting transcripts, highlights, and speakers.
  • Optional native recordings routed directly into Otter.

Orchestration

  • Google Opal (or comparable) for multi-step agent execution.
  • Zapier / n8n / Make as event-driven glue across Airtable, Docs, calendars, and CMS.
  • FastAPI microservice for custom webhooks or long-running jobs.

Storage & tracking

  • Airtable as the project hub (Inputs, Artifacts, Distribution, Tasks, Time Logs).
  • Google Drive & Docs for living drafts.
  • MongoDB (optional) for transcript retrieval and RAG.

AI writing & editing

  • OpenAI for drafting, structuring, summaries, and titles.
  • Hemingway + Grammarly for readability and tone checks.
  • Midjourney / DALL·E for creative assets; Surfer SEO / Clearscope for search insights.

Publishing

  • LinkedIn, Medium, Substack, Ghost as distribution channels.
  • Buffer / Hypefury for cross-posting and scheduling.

Analytics

  • Airtable Interfaces for quick views and monitoring.
  • Metabase / Looker Studio (optional) for deeper analysis and reporting.

Airtable data model

Tables that anchor every run, artifact, and distribution touchpoint.

Each table tracks a different slice of the workflow—from raw inputs to publishing metrics—so you can audit progress, ownership, and quality in seconds.

Inputs

Source of truth for meetings & notes

  • Input ID, Source, Otter Link, Title, Owner, Project
  • Raw Transcript, Highlights, Status, Tags, Created At

Checklists

Structured playbooks

  • Checklist ID, Name, Category, Steps (JSON)
  • SLA (hrs), Default Owner, Version, HTML Path, Active?

Runs & Run Steps

Execution telemetry

  • Run ID, Checklist ID, Input ID, Assignee, Status
  • Start/End Time, Total Minutes, Step Count, Artifacts
  • Run Steps include tool, prompt reference, inputs, outputs, duration, status, reruns.

Artifacts & Distribution

Outputs & publishing

  • Artifact ID, Type, Format, Storage Link, Version, Approval
  • Distribution covers channel, post URL, schedule, performance metrics.

Prompts & Timesheets

Governance & time tracking

  • Prompt metadata: purpose, template, inputs schema, model, owner, version.
  • Timesheet entries: person, run/task link, start/end, minutes, notes.

Operational checklists

Standard operating procedures to move from capture to publish.

Each checklist is versioned, time-bound, and tied back to prompts so your VA or agent knows exactly what “done” means.

Meeting → Insight Brief

Otter to executive summary in 7 precise steps.

  1. Ingest transcript via Zapier webhook and log highlights in Inputs.
  2. Run the MEETING_STRUCTURER prompt to convert notes into agenda, decisions, owners, risks.
  3. Generate a 200–300 word executive brief with EXEC_SUMMARY into Google Docs.
  4. Push action items into Airtable tasks with owners and due dates.
  5. Create a LinkedIn-ready distribution summary via OpenAI.
  6. Quality pass through Hemingway & Grammarly to confirm readability ≤ Grade 7.
  7. Mark the run complete; automation stamps duration and completion metrics.

Blog Refinement & SEO

Transform a draft into a publish-ready article with search coverage.

  1. Capture draft intake in Inputs.
  2. Outline sections and estimated word counts.
  3. Produce an SEO_BRIEF with keywords, SERP gaps, and questions.
  4. Draft V1 via BLOG_DRAFTER including TL;DR, headings, and CTAs.
  5. Create visuals with DALL·E or Midjourney and log artifacts.
  6. Polish readability to Grade 6–8 and finalize metadata.
  7. Publish to blog/Medium/Substack, cross-post to LinkedIn, and record distribution metrics.
  8. Run a retrospective noting minutes, blockers, improvements.

Dashboard design

High-signal interface to manage runs, outputs, and publishing cadence.

The dashboard surfaces throughput, time load, and quality indicators with zero clutter. Build first in Airtable Interfaces, then extend into a custom web app.

Header KPIs

  • Runs in progress or due today.
  • Average minutes per checklist (7-day & 30-day).
  • Output velocity (artifacts per week) and publish rate.
  • Quality flags for readability and approvals.

Operational views

  • Pipeline board grouped by checklist: New → In Progress → Review → Done.
  • Run detail panel with steps, timestamps, artifacts, blockers, SLA countdown.
  • Publishing calendar showing scheduled posts with channel icons.
  • SEO panel highlighting last 10 posts, keyword targets, SERP rank placeholder, impressions.

Time & metrics

  • Timesheet heatmap by person and project.
  • Total minutes logged today and this week.
  • Checklist efficiency formula: Total Minutes ÷ Completed Steps.
  • Publish rate: Published ÷ Scheduled.

Prompt library

Reusable instructions keep outputs consistent and auditable.

Version each prompt, attach it to run steps, and log the model + parameters used. Highlights from the core library:

MEETING_STRUCTURER

Extract purpose, agenda, decisions, owners, deadlines, risks from transcripts in JSON.

EXEC_SUMMARY

200–300 word executive brief with context, key decisions, risks, and next steps.

SEO_BRIEF

Keyword plan, SERP gap analysis, questions to answer, competitor review, titles, and meta descriptions.

BLOG_DRAFTER

1,200–1,800 word post referencing the SEO brief with TL;DR, CTAs, and image placeholders.

SOCIAL_SNIPPETS

Multi-channel social copy package: LinkedIn, X thread, newsletter blurb.

Security & governance

Protect sensitive data while keeping automations flexible.

  • Store API keys in Opal or a secrets manager, never directly in Airtable.
  • Mask PII in transcripts before sending to external models; maintain an allowlist.
  • Version prompts and attach Prompt ID + version to each artifact.
  • Require human-in-the-loop approvals before distribution.
  • Log errors: failed steps flag the run as blocked and notify the assignee.
  • Retain audit trails via run logs, artifacts history, and publishing records.
