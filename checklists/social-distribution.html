Social Distribution Launch

Category: Distribution • SLA: 4h • Version: 1.0

Transform approved artifacts into platform-native posts, schedule distribution, and capture performance signals.

Inputs Required

  • Approved artifact link (blog, brief, or asset)
  • Canonical URL + UTM parameters
  • Distribution channel list with posting windows
  • Prompt reference: SOCIAL_SNIPPETS
  • Brand voice + compliance checklist

Steps

1. Confirm artifact approval Tool: Airtable Artifacts • SLA: 5 min
Artifact status = Approved, linked runs verified, and publishing notes captured.
2. Generate channel copy Agent: OpenAI • Prompt: SOCIAL_SNIPPETS • SLA: 12 min
LinkedIn post, X thread, and newsletter blurb created with canonical URL placeholder.
3. Localize per channel Tool: Manual edit + Grammarly • SLA: 15 min
Voice/tone check complete, hashtags reviewed, CTA consistent, compliance cleared.
4. Schedule posts Tool: Buffer / Hypefury • SLA: 15 min
Posts scheduled with correct times, images, and shortened URLs; Distribution rows created per channel.
5. Update Airtable Distribution Automation: Zapier sync • SLA: 5 min
Distribution table populated with scheduled timestamp, platform, and post URL placeholders.
6. Monitor publishing Tool: Buffer analytics • SLA: 30 min (rolling)
Published time recorded, post URLs confirmed live, initial impressions/clicks logged after 24h.
7. Feedback loop Tool: Airtable notes • SLA: 8 min
Key learnings, performance highlights, and follow-up experiments documented in Run.
8. Close checklist Automation: Airtable update • SLA: 3 min
Run status set to Done, minutes logged, metrics forwarded to dashboard.

Outputs & Quality Gates

  • Channel-specific copy deck (LinkedIn, X, newsletter)
  • Scheduled posts with assets attached
  • Distribution table entries with UTMs + tracking tags
  • Initial performance metrics logged for dashboard ingest
  • Run retrospective captured with optimization ideas
